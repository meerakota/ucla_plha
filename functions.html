Functions#

The functions available in the ucla_plha package are summarized below, followed by a detailed description of each. Some general notes on the functions: - \(N\) = number of events - \(L\) = number of FSL or intensity measure values at which to compute hazard curves - \(M\) = number of magnitude bins for disaggregation - \(R\) = number of distance bins for disaggregation - \(E\) = number of epsilon bins for disaggregation

decompress_ucerf3_source_data()#

Some of the input UCERF3 source-model files are quite large and are compressed as NumPy archives to keep the package installation efficient. Within the ucerf3_fm31 and ucerf3_fm32 directories under source_models/fault_source_models, the ruptures.npz and ruptures_segments.npz files are compressed as (.npz) NumPy archives. An .npz file is a NumPy specific zip archive format used to save multiple NumPy arrays into a single file on disk. Loading the compressed archives at run time is slower than loading uncompressed data, so this function reads the .npz archives for both UCERF3 branches and writes uncompressed .npy versions. The ucla_plha code checks for the .npy files first and uses them if present, otherwise falling back to the .npz archives. It only needs to be run once.

get_source_data(source_type, source_model, p_xyz, dist_cutoff, gmms)#

This function takes inputs of the type of source data (fault or point source), the source model (branch dependent), the site of interest, distance cutoff (e.g. 200 km), and the ground motion models of interest. For fault and point source data it returns filtered values (given distance cutoff) for magnitude, fault type (dependent on rake), rate, \(R_{JB}\) (Joyner-Boore distance in km), \(R_{rup}\) (rupture distance in km), \(R_x\) (the horizontal distance from the top edge of the rupture in km), \(R_{x1}\) (in km), \(R_{y0}\) (in km), dip (perpendicular to strike, in degrees), \(Z_{tor}\) (depth to top of rupture in km), and \(Z_{bor}\) (depth to bottom of rupture in km).

Returns magnitude, fault type, rate, distance, and fault geometry terms.

Parameters:

  • source_type (str) – Either "fault_source_models" or "point_source_models".

  • source_model (str) – Directory for the source model within the source-type directory. Currently either "ucerf3_fm31" or "ucerf3_fm32".

  • p_xyz (numpy.ndarray of float) – Array containing the x, y, z coordinates of the point of interest, length 3.

  • dist_cutoff (float) – Maximum distance to consider in the seismic hazard analysis.

  • gmms (array of str) – Ground motion models to use, one or more of "ask14", "bssa14", "cb14", "cy14".

Returns:

A tuple of arrays, each of length \(N\):

  • m – magnitudes.

  • fault_type – fault types (1 = reverse, 2 = normal, 3 = strike slip).

  • rate – annual rate of occurrence of each event.

  • rjb – \(R_{JB}\), Joyner-Boore distance between the site and each rupture, in km.

  • rrup – \(R_{rup}\), rupture distance between the site and each rupture, in km.

  • rx – \(R_x\), distance to the surface projection of the top of each rupture, measured perpendicular to strike, in km.

  • rx1 – \(R_{x1}\), distance to the surface projection of the bottom of each rupture, measured perpendicular to strike, in km.

  • ry0 – \(R_{y0}\), distance to the surface projection of each rupture, measured parallel to strike, in km.

  • dip – dip angle of each rupture, in degrees.

  • ztor – \(Z_{tor}\), depth to the top of each rupture, in km.

  • zbor – \(Z_{bor}\), depth to the bottom of each rupture, in km.

get_ground_motion_data(gmm, vs30, fault_type, rjb, rrup, rx, rx1, ry0, m, ztor, zbor, dip, z1p0, z2p5, measured_vs30)#

Computes arrays containing the mean and standard deviation of the natural log of a ground motion intensity measure.

Parameters:

  • gmm (str) – Ground motion model. One of "ask14", "bssa14", "cb14", "cy14".

  • vs30 (float) – \(V_{S30}\), time-averaged shear-wave velocity in the upper 30 m, in m/s.

  • measured_vs30 (bool) – Whether \(V_{S30}\) is measured (True) or inferred (False).

  • z1p0 (float) – \(Z_{1.0}\), depth to the 1.0 km/s shear-wave isosurface, in km.

  • z2p5 (float) – \(Z_{2.5}\), depth to the 2.5 km/s shear-wave isosurface, in km.

  • fault_type – Fault type for each rupture (1 = reverse, 2 = normal, 3 = strike slip), length \(N\).

  • rjb – \(R_{JB}\), Joyner-Boore distances, length \(N\).

  • rrup – \(R_{rup}\), rupture distances, length \(N\).

  • rx – \(R_x\), distance to the surface projection of the top of each rupture, perpendicular to strike, length \(N\).

  • rx1 – \(R_{x1}\), distance to the surface projection of the bottom of each rupture, perpendicular to strike, length \(N\).

  • ry0 – \(R_{y0}\), distance to the surface projection of each rupture, parallel to strike, length \(N\).

  • m – Magnitudes, length \(N\).

  • ztor – \(Z_{tor}\), depth to the top of each rupture, in km, length \(N\).

  • zbor – \(Z_{bor}\), depth to the bottom of each rupture, in km, length \(N\).

  • dip – Dip angle of each rupture, in degrees, length \(N\).

Returns:

  • mu_ln_pga – \(\mu_{\ln PGA}\), mean of the natural log of the ground motion intensity measure, length \(N\).

  • sigma_ln_pga – \(\sigma_{\ln PGA}\), standard deviation of the natural log of the intensity measure, length \(N\).

get_liquefaction_cdfs(m, mu_ln_pga, sigma_ln_pga, fsl, liquefaction_model, config)#

Computes log-normal cumulative distribution functions for the factor of safety of liquefaction.

Parameters:

  • m – Magnitudes, length \(N\).

  • mu_ln_pga – \(\mu_{\ln PGA}\), mean of the natural log of the ground motion intensity measure, length \(N\).

  • sigma_ln_pga – \(\sigma_{\ln PGA}\), standard deviation of the natural log of the intensity measure, length \(N\).

  • fsl – Factor-of-safety values at which to compute the liquefaction hazard curve, length \(L\).

  • liquefaction_model (str) – One of "cetin_et_al_2018", "moss_et_al_2006", "boulanger_idriss_2016", "boulanger_idriss_2012", "ngl_smt_2024".

  • config (dict) – Dictionary read from the (case-insensitive) config file.

Returns:

  • fsl_cdfs – cumulative distribution functions representing the down-crossing rate of \(FS_L\) for each event, shape \(N \times L\).

  • eps – \(\varepsilon\) values (number of standard deviations of \(\ln(FS_L)\) relative to its mean), shape \(N \times L\).

get_disagg(hazards, m, r, eps, m_bin_edges, r_bin_edges, eps_bin_edges)#

This function calculates the seismic and liquefaction hazard disaggregation (i.e., how different earthquake scenarios (magnitude, distance, etc.) contribute to the overall seismic and liquefaction hazard at a specific location). It takes NumPy arrays of hazard values (size of number of IM values x number of events), magnitude values, distance values, \(\varepsilon\) values, and lastly magnitude, distance, and \(\varepsilon\) bin edges. It returns disaggregation values for the desired bins.

Parameters:

  • hazards – Hazard values, shape \(L \times N\).

  • m – Magnitude values, length \(N\).

  • r – Distance values, length \(N\).

  • eps – \(\varepsilon\) values, shape \(L \times N\).

  • m_bin_edges – Magnitude bin edges, length \(M + 1\).

  • r_bin_edges – Distance bin edges, length \(R + 1\).

  • eps_bin_edges – \(\varepsilon\) bin edges, length \(E + 1\).

Returns:

disagg – contribution to the hazard within each magnitude, distance, and \(\varepsilon\) bin, for each PGA (PSHA) or \(FS_L\) (PLHA) value, shape \(L \times M \times R \times E\).

get_hazard(config_file)#

Read a config file and run the PSHA and/or PLHA. This is the main entry point of the package.

Parameters:

config_file (str) – Filename, including path, of the config file. Must follow the schema defined by ucla_plha_schema.json (see Config File Format). The config file is case-insensitive.

Returns:

output – dictionary containing the analysis output. It preserves all of the inputs and adds the hazard curve(s) and any requested disaggregation data.
